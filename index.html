Recorrido sugerido

Podés empezar por ejercicios de lectura, seguir con corrección y completar, y dejar la creación desde cero para el final. No hace falta resolver todo en orden, pero sí conviene avanzar de lo más guiado a lo más autónomo.

1. Entrada suave

Empezá por interpretar programas ya hechos. El foco está en entender qué hacen y cómo están armados.

2. Ajuste fino

Después pasá a corregir y completar código. Ahí aparece más claramente si recordás sintaxis y lógica.

3. Autonomía

Terminá con programas breves desde cero. No tienen trampa: son problemas simples para recuperar confianza.

A · Interpretar Mínimo viable

1. ¿Qué hace este programa?
  1. Explicá qué problema resuelve el programa.
  2. Indicá qué valor mostrará en pantalla.
  3. Decí para qué sirve la variable mayor.
  4. Explicá qué función cumple el for.
public class Ejercicio1 {
    public static void main(String[] args) {
        int[] numeros = {12, 7, 25, 9, 18};
        int mayor = numeros[0];

        for (int i = 1; i < numeros.length; i++) {
            if (numeros[i] > mayor) {
                mayor = numeros[i];
            }
        }

        System.out.println("El mayor es: " + mayor);
    }
}

public class Ejercicio3 {

        for (int i = 0; i < numeros.length; i++) {

Cómo trabajar con esta página

La idea es que puedas usar este banco de ejercicios de dos maneras: primero para practicar en casa y reactivar recuerdos; después, para retomar algunos en clase y ver qué conviene reforzar.

En casa

Elegí al menos un ejercicio de cada categoría. Anotá dudas, intentos y errores. No pasa nada si no sale a la primera.

En clase

Algunos de estos ejercicios volverán a aparecer. La meta será explicar lo que hiciste, no solo mostrar el resultado.

Consejo

Si te bloqueás, empezá por decir con palabras qué debería hacer el programa. Pensar antes de escribir ayuda mucho.
